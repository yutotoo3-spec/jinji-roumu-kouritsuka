# 入社手続き管理ツール（jinji-roumu-kouritsuka）

**採用確定（HERP）→ 契約締結 → 社宅確認 → 基礎情報回収 → 入社手続き（マネーフォワード等のアカウント発行）→ 労務手続き → 入社後フォロー** までの一連の業務プロセスを、一気通貫で・漏れなく管理するための社内ツールです。正社員と業務委託の両方に対応しています。

## できること

### 管理の基本
- **雇用区分別の標準タスク自動生成** — 正社員（6フェーズ・30件）/ 業務委託（4フェーズ・14件）それぞれ専用のタスクが、入社日（稼働開始日）から逆算した期日付きで自動生成されます
- **HERP Webhook自動連携** — HERPで採用決定した瞬間に、登録とタスク生成が自動で行われます（重複は自動スキップ、雇用区分も判定）
- **社宅の条件分岐** — 正社員のみ。「利用しない」にすると社宅タスクは自動で対象外になります。業務委託には社宅・社会保険等のタスクはそもそも生成されません
- **期限管理** — 期限超過・期日間近（3日以内）をダッシュボードで警告。法定期限（社会保険は入社5日以内など）も組み込み済み
- **入社日変更への追従** — 入社日を変更すると未完了タスクの期日だけが自動再計算されます

### 効率化機能
- **本人ページ（ポータル）** — 入社者・委託メンバーごとに専用URLを発行。本人は自分の提出物だけをチェックリストで確認でき、住所・連絡先・振込口座はフォームで提出。提出と同時に人事側のタスクが自動完了します（マイナンバーは安全のため入力不可の設計）
- **Slackデイリー通知** — `SLACK_WEBHOOK_URL` を設定すると、毎朝9時台に「期限超過」「本日期日」「社宅未確認」をSlackに自動投稿します
- **メール文面の差し込み生成** — 入社案内・提出リマインド・社宅案内・契約手続き案内を、氏名・期日・提出物一覧・本人ページURL入りで自動生成。コピーしてすぐ送れます
- **マネーフォワード取込CSV** — 本人から回収した基礎情報を含むCSVを1クリックでダウンロード。二重入力をなくします
- **入社後フォロー** — 正社員は30/60/90日面談・試用期間評価、業務委託は稼働振り返り・契約更新期日までタスク化

## 使い方

Node.js 20以上が必要です。依存パッケージはありません。

```bash
# サーバー起動（http://localhost:3000）
npm start

# デモデータを投入して動作確認する場合
npm run seed
npm start

# テスト
npm test
```

### 環境変数

| 変数 | 用途 |
|---|---|
| `PORT` | ポート番号（既定: 3000） |
| `DATA_DIR` | ファイル保存時のデータ保存先（既定: `./data`） |
| `UPSTASH_REDIS_REST_URL` / `UPSTASH_REDIS_REST_TOKEN` | 両方設定するとデータを Upstash Redis（無料枠あり）に保存。**永続ディスクのない無料ホスティングで必須** |
| `BASIC_AUTH_USER` / `BASIC_AUTH_PASSWORD` | 両方設定すると人事画面・APIにBasic認証がかかる。**インターネット公開時は必須**。本人ページ（`/portal/*`）とWebhookは認証対象外のまま動きます |
| `HERP_WEBHOOK_TOKEN` | Webhookの認証トークン（設定を推奨） |
| `SLACK_WEBHOOK_URL` | SlackのIncoming Webhook URL。設定すると毎朝ダイジェストを自動送信 |
| `DIGEST_HOUR` | Slack通知を送る時刻（既定: 9） |

## インターネットへの公開（デプロイ）

チーム利用・HERP Webhook・本人ページ・Slack通知を実際に動かすには、サーバーの公開が必要です。`Dockerfile`・`render.yaml`（Render用）・`railway.json`（Railway用）を同梱しています。

### 無料構成（月0円）: Render Free + Upstash Redis Free

Render の無料プランは永続ディスクが使えないため、データ保存に Upstash Redis の無料枠を組み合わせます。

**手順1: Upstash（データ保存先・無料）**

1. https://upstash.com にサインアップ（GitHubアカウントでOK）
2. **Create Database** → リージョンは Japan/Tokyo（近いもの）を選択 → 作成
3. データベース詳細画面の **REST API** セクションにある `UPSTASH_REDIS_REST_URL` と `UPSTASH_REDIS_REST_TOKEN` を控える

**手順2: Render（ホスティング・無料）**

1. https://render.com にサインアップし、GitHubアカウントを連携
2. ダッシュボード → **New → Blueprint** → このリポジトリを選択（`render.yaml` が自動で読み込まれます）
3. 環境変数の入力を求められたら以下を設定:
   - `BASIC_AUTH_USER` / `BASIC_AUTH_PASSWORD` — 人事画面のログインID・パスワード
   - `UPSTASH_REDIS_REST_URL` / `UPSTASH_REDIS_REST_TOKEN` — 手順1で控えた値
4. 作成完了後、発行されたURL（`https://～.onrender.com`）を開き、Basic認証でログイン

**手順3: 毎朝のSlack通知（無料cronで起動）** ※Slack通知を使う場合のみ

Render Freeは15分アクセスがないとスリープするため、アプリ内蔵のスケジューラは当てになりません。無料の外部cronから毎朝叩きます。

1. Slackで Incoming Webhook を作成し、URLを Render の環境変数 `SLACK_WEBHOOK_URL` に設定
2. https://cron-job.org （無料）にサインアップし、ジョブを作成:
   - URL: `https://<あなたのURL>/api/notify/slack?token=<HERP_WEBHOOK_TOKENの値>`（トークンはRenderの環境変数画面で確認）
   - Request method: **POST**、スケジュール: 毎日 9:00 など

**無料構成の注意点**

- 15分アクセスがないとスリープし、次に開くとき（本人ページ含む）**起動に1分ほど**かかります
- Upstash無料枠は256MB・50万コマンド/月で、このツールの用途（数十〜数百名分のJSON）には十分です

### 有料構成（月$7〜）: Render Starter

スリープなし・外部サービス不要にしたい場合は、`render.yaml` のコメントに従って `plan: starter` + 永続ディスクに切り替えてください（Upstash・外部cron不要になります）。

### Railway の場合（月$5〜）

1. https://railway.app にサインアップし、**New Project → Deploy from GitHub repo** でこのリポジトリを選択
2. サービスの **Settings → Volumes** でボリュームを追加し、Mount Path を `/data` にする
3. **Variables** で `BASIC_AUTH_USER` / `BASIC_AUTH_PASSWORD` / `HERP_WEBHOOK_TOKEN` / `DATA_DIR=/data` を設定
4. **Settings → Networking → Generate Domain** で公開URLを発行

### デプロイ後にやること

1. 発行されたURLにアクセスし、Basic認証で入れることを確認
2. **HERP連携**: HERPのWebhook設定に `https://<あなたのURL>/api/webhook/herp?token=<HERP_WEBHOOK_TOKENの値>` を登録
3. **Slack通知のテスト**: `curl -X POST "https://<あなたのURL>/api/notify/slack?token=<トークン>"` で即時送信して確認
4. 本人ページのURL（詳細画面からコピー）が `https://` の公開URLになっていることを確認して案内メールに使う

## 業務フロー（テンプレート）

### 正社員（6フェーズ・30タスク）

| フェーズ | 主なタスク | 期日の目安 |
|---|---|---|
| ① 契約締結 | 内定通知書 → 雇用契約書 → 電子契約 → 締結確認 | 入社30〜14日前 |
| ② 社宅 | 利用確認 →（利用時のみ）物件手配・誓約書・給与控除 | 入社21〜3日前 |
| ③ 基礎情報回収 | 住所・口座・マイナンバー・年金/雇用保険番号・扶養申告書・健診書 | 入社14〜3日前 |
| ④ 入社手続き | マネーフォワード / Google Workspace / Slack 発行、PC・入館証 | 入社7〜3日前 |
| ⑤ 労務手続き | 社会保険（**入社5日以内**）、雇用保険（**翌月10日**）、住民税、法定三帳簿 | 入社日〜10日後 |
| ⑥ 入社後フォロー | 30/60日面談、試用期間評価・本採用判断、90日振り返り | 入社30〜90日後 |

### 業務委託（4フェーズ・14タスク）

| フェーズ | 主なタスク | 期日の目安 |
|---|---|---|
| ① 契約締結 | 業務範囲・単価の確定、反社チェック、業務委託契約書・NDA | 開始18〜7日前 |
| ② 支払設定 | インボイス登録番号の確認、振込口座、MFへの支払先登録 | 開始7〜3日前 |
| ③ 受け入れ準備 | セキュリティ誓約、アカウント発行（最小権限）、キックオフ | 開始5日前〜当日 |
| ④ 契約管理 | 稼働30日振り返り、**契約更新・終了の判断**、権限棚卸し | 開始30〜75日後 |

タスクの内容・担当・期日は [`template.json`](template.json) で管理しています。自社の運用に合わせて編集してください（編集後に登録した人から反映されます）。

## HERPとの連携

### 方法1: Webhook自動連携（推奨・手間なし）

HERP側のWebhook（外部システム通知）に本ツールのURLを設定すると、採用決定時に自動で登録されます。

```
POST http://<このツールのホスト>:3000/api/webhook/herp?token=<トークン>
```

- ペイロードのキー名は柔軟に解釈します（英語/日本語キー、`candidate` 等のネスト、`雇用形態: 業務委託` の判定にも対応）
- 形式が合わない場合は `lib/store.js` の `WEBHOOK_KEY_ALIASES` にキー名を追加してください

### 方法2: 画面から手動登録

「＋ 新規登録」から雇用区分を選んで30秒で登録できます。

## 本人ページについて

- URLは `http://<ホスト>/portal/<本人ごとのトークン>`。各人の詳細画面からコピーできます
- 本人に見えるのは自分の提出物・期日のみで、人事用メモ・HERP URL・他の人の情報は見えません
- 提出された基礎情報は詳細画面に表示され、MF取込CSVにも反映されます

## 構成

```
server.js          # HTTPサーバー（API + 静的配信 + Slack通知スケジューラ）
lib/store.js       # データ永続化・タスク生成・Webhook・ポータル・通知ロジック
template.json      # 業務プロセスのマスタ（雇用区分ごとのフェーズ・タスク・期日）
public/index.html  # 人事向け画面（SPA）
public/portal.html # 本人ページ（スマホ対応）
scripts/seed.js    # デモデータ投入
test/              # ユニットテスト（node --test）
data/              # データ保存先（gitignore済み・自動生成）
```

## 運用上の注意

- **マイナンバーそのものはこのツールに保存しない運用**を前提にしています（本人ページにも入力欄を設けていません）
- 人事向け画面は認証なしです。社内ネットワーク・VPN内での利用を想定しており、インターネットに公開する場合はBasic認証等の追加が必要です
- 本人ページはトークン付きURLを知っている本人だけがアクセスできる想定です。URLの取り扱いには注意してください
