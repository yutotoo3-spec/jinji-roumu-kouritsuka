# 入社手続き管理ツール（jinji-roumu-kouritsuka）

正社員の**採用確定（HERP）→ 契約締結 → 社宅確認 → 基礎情報回収 → 入社手続き（マネーフォワード等のアカウント発行）→ 労務手続き**までの一連の業務プロセスを、一気通貫で・漏れなく管理するための社内ツールです。

## できること

- **入社者ごとの標準タスク自動生成** — 入社者を登録すると、契約締結から労務手続きまでの標準タスク（26件）が、入社日から逆算した期日付きで自動生成されます
- **HERP CSVインポート** — HERPからエクスポートした候補者CSVを取り込んで一括登録できます（重複は自動スキップ）
- **社宅利用の条件分岐** — 社宅利用「未確認 / 利用する / 利用しない」を管理。「利用しない」にすると社宅関連タスクは自動で対象外になり、進捗率からも除外されます
- **期限管理** — 期限超過・期日間近（3日以内）のタスクをダッシュボードで警告。法定期限（社会保険は入社5日以内など）もタスクに組み込み済み
- **進捗の見える化** — 入社者ごとの進捗率、フェーズごとの完了状況、フィルタ（入社前 / 期限超過あり / 完了）
- **入社日変更への追従** — 入社日を変更すると、未完了タスクの期日だけが自動で再計算されます

## 使い方

Node.js 20以上が必要です。依存パッケージはありません。

```bash
# サーバー起動（http://localhost:3000）
npm start

# デモデータを投入して動作確認する場合
npm run seed
npm start

# テスト
npm test
```

ポートを変える場合は `PORT=8080 npm start`、データ保存先を変える場合は `DATA_DIR=/path/to/dir npm start`。

## 業務フロー（フェーズ構成）

| フェーズ | 主なタスク | 担当 | 期日の目安 |
|---|---|---|---|
| ① 契約締結 | 内定通知書送付 → 雇用契約書作成 → 電子契約送信 → 締結確認 | 人事 | 入社30〜14日前 |
| ② 社宅 | 利用有無の確認 →（利用時のみ）物件手配・誓約書・給与控除設定 | 人事・労務 | 入社21〜3日前 |
| ③ 基礎情報回収 | 住所・口座・マイナンバー・年金/雇用保険番号・扶養申告書・健診書 等 | 本人 | 入社14〜3日前 |
| ④ 入社手続き | マネーフォワード（給与・勤怠・経費）/ Google Workspace / Slack のアカウント発行、PC・入館証手配 | 労務・情シス | 入社7〜3日前 |
| ⑤ 労務手続き | 社会保険資格取得（**入社5日以内**）、雇用保険資格取得（**翌月10日まで**）、住民税切替、法定三帳簿 | 労務 | 入社日〜10日後 |

タスクの内容・担当・期日オフセットは [`template.json`](template.json) で管理しています。自社の運用に合わせて自由に編集してください（編集後に登録した入社者から反映されます）。

## HERPとの連携

### 方法1: Webhook自動連携（推奨・手間なし）

HERP側のWebhook（外部システム通知）に本ツールのURLを設定すると、採用決定時に自動で入社者が登録され、標準タスクが生成されます。

```
POST http://<このツールのホスト>:3000/api/webhook/herp?token=<トークン>
```

- トークン認証を有効にするには `HERP_WEBHOOK_TOKEN=<任意の文字列> npm start` で起動します（`?token=` またはリクエストヘッダー `X-Webhook-Token` で照合）
- ペイロードのキー名は柔軟に解釈します：氏名（`name` / `candidate_name` / `氏名` 等）、入社日（`join_date` / `hire_date` / `入社予定日` 等）、`candidate` / `data` などのネストにも対応
- 登録済みの人（メールアドレス一致）は重複として登録されません（200 OK を返すのでHERP側のリトライも発生しません）
- HERP側のペイロード形式が合わない場合は `lib/store.js` の `WEBHOOK_KEY_ALIASES` にキー名を追加してください

curlでの動作確認：

```bash
curl -X POST 'http://localhost:3000/api/webhook/herp' \
  -H 'Content-Type: application/json' \
  -d '{"name":"山田 太郎","email":"taro@example.com","position":"エンジニア","join_date":"2026-09-01"}'
```

### 方法2: CSVインポート（一括登録）

HERPの候補者一覧からCSVをエクスポートし、画面右上の「HERP CSVインポート」から取り込みます。

- 必須列：`氏名`、`入社日`（`入社予定日` も可。`2026/8/1`・`2026-08-01`・`2026年8月1日` いずれの形式もOK）
- 任意列：`メールアドレス`、`ポジション`（`応募ポジション` も可）、`部署`
- メールアドレスが同じ（無い場合は氏名+入社日が同じ）既存データは重複としてスキップされます

### 方法3: 画面から手動登録

1名だけ追加する場合は「＋ 入社者を登録」から30秒で登録できます。

## 構成

```
server.js          # HTTPサーバー（API + 静的配信、依存パッケージなし）
lib/store.js       # データ永続化・タスク生成・CSVインポートのロジック
template.json      # 業務プロセスのマスタ（フェーズ・タスク・担当・期日オフセット）
public/            # フロントエンド（SPA）
scripts/seed.js    # デモデータ投入
test/              # ユニットテスト（node --test）
data/db.json       # データ保存先（gitignore済み・自動生成）
```

## 運用上の注意

- データは `data/db.json` に保存されます。**マイナンバーそのものはこのツールに保存しない運用**を推奨します（回収タスクの完了チェックのみ管理）
- 社内ネットワーク・VPN内での利用を想定した認証なしのツールです。外部公開しないでください
